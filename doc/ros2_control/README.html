Installation

ros2_control is now released for galactic:

$ sudo apt install ros-galactic-ros2-control
$ sudo apt install ros-galactic-ros2-controllers
# maybe also control-toolbox?

Workflow

Usually the main packages are:

  1. description package, store URDF description files, rviz configurations and meshes

  2. bringup package, holds launch files and runtime configurations for robots

  3. hardware package, implements the hardware interfaces

  4. controllers package, implements custom controllers.

The workflow can be thought as follows:

  1. Implement the description package. It contains:

    • The robot URDF (XACRO) defining the robot structure and physical properties.

    • The ros2_control URDF (XACRO) defining the logical components of the robot and how they’re managed by ros2_control framework.

  2. Implement the hardware package, containing the hardware resources that will be requested by the controllers and managed by the RM.

  3. Implement the controllers, either in the bringup package or in a more specific package for controllers if they’re built. It contains:

    • The controllers YAML config file, declaring the CM and controllers configuration (i.e. update rate, controllers to be activated and their parameters…)

    • optionally, write custom controllers if they’re not provided by ros2_controllers package.

  4. Implement the bringup package, with launch files starting CM node and controllers using a default ros2_control node
